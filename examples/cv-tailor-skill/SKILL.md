# CV Tailor Skill

## When to Use This Skill

Activate when the user:
- Shares a job description and asks for CV help
- Wants to tailor their resume for a specific role
- Asks "how should I update my CV for this job?"

## Knowledge

### What Recruiters Look For
- **Keyword match** — ATS systems scan for exact terms from the job description
- **Relevant experience first** — Move matching skills and projects to the top
- **Quantified impact** — "Improved API response time by 40%" beats "Worked on APIs"
- **Honest framing** — Reframe existing experience, never invent what isn't there

### Common Student Mistakes
1. Using one generic CV for every application
2. Listing technologies without context (e.g., "Python" vs. "Built a REST API in Python")
3. Burying relevant coursework or projects below irrelevant work experience
4. Missing keywords the job description explicitly asks for

## Workflow

When the user provides a job description:
1. Identify the key skills, technologies, and qualifications the employer wants
2. Ask for their CV if not provided
3. Compare the job requirements against the CV
4. Suggest specific changes:
   - Which bullet points to reword
   - Which skills to move up
   - What to add or remove
5. Rewrite the relevant CV sections with improvements applied
6. Explain **why** each change makes the CV stronger for this role

## Rules

- Keep suggestions practical and honest — don't invent experience the student doesn't have
- Focus on better framing of existing skills and projects
- If the student lacks a key requirement, suggest how to address it (course, side project, etc.)
- Always preserve the student's authentic voice
